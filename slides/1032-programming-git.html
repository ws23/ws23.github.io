作業練習

軟體安裝、環境設定

請參考前面教學投影片、教學網站

repo clone

  • 把專案repo整個抓下來
  • git clone https://github.com/ws23/waterball-fighting

create new branch

  • 新增一個屬於你自己用的branch
  • 轉移到該branch

edit file

  • 編輯 practice.txt
  • 記得在上面打上你的ID或名字，方便讓我知道你有改上去
  • 記得不要把別人的文字砍掉了，版本控制是留有記錄的科科

add to staging area

  • 把編輯好的檔案加入staging area
  • git add practice.txt

commit to repository

  • 把staging area中的檔案提交至版本控制庫
  • git commit -m '提交訊息，請寫有意義的註解:p'

push to remote

  • 把版本控制庫的檔案丟到遠端
  • git push
  • git push -u origin branch_name

merge branch

  • git checkout master 回到主branch
  • git merge branch_name 合併你前面有做修改的branch

pull to local

  • 把遠端版本控制庫的資料拉到本地端
  • git pull
  • 如果有別人更新遠端的話，可以用這個把本地與遠端同步

完成作業練習

總結常用步驟(不用branch)

  1. git pull
  2. 編輯
  3. git add
    • git rm // 把檔案從staging area拉回工作區域
  4. git commit -m 'message'
  5. git push

總結常用步驟(branch)

  1. git branch XDD // 建另一個叫做XDD的branch
  2. git checkout XDD // 轉移到XDD的branch
  3. 各種修改測試可以多add多commit直到完整可以合併
  4. git checkout master // 轉移回到主branch
  5. git merge XDD // 如果沒有衝突要解決就會直接完成

其他常用動作

  • git status
  • git diff
  • git log
  • git checkout
  • git reset
